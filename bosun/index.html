The problem

Convention is invisible to an unbriefed agent

PressGang's best features leave barely a trace in a child theme's code — controllers route by convention, config merges silently, context getters wire themselves. Great for developers who know the ship; invisible to AI agents who don't.

An unbriefed agent writes working code that misses everything the framework gives you: hand-rolled query arrays, stub template files, hooks in functions.php. Bosun closes that gap at the source.

Composed from the ship's manifest

Not a static template pasted into every repo — Bosun inspects each theme and writes guidance for what's actually there.

📋 Inventory-driven

Detects installed packages, versions, and enabled features — template routing, Quartermaster, WooCommerce — and briefs accordingly.

🧠 Every agent format

Writes CLAUDE.md and AGENTS.md, plus agent skills — one briefing, every AI coding tool aboard.

🔄 Re-run anytime

Add a package, enable a feature, re-run wp bosun install — the briefing keeps pace with the ship.

🔌 Live tools, not just docs

Where Capstan can serve MCP, Bosun wires its introspection — routing, config, health, version-aware docs search — straight into your editor's .mcp.json, so agents observe the live theme, not just read about it.
